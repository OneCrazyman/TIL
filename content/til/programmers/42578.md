### 첫번째 제출코드
```cs
// 프로그래머스 - 해시 - 의상

// 각 종류별 하나씩 꺼내서 조합한다 -> 경우의 수 (안꺼내는것도 포함하니, 총갯수+1씩 곱할것)
// 해시셋으로 각 sort별 리스트를 생성하고 -> 리스트에 옷을 add
// 마지막으로 해시셋 순회 -> 경우의 수 계산

using System;
using System.Collections.Generic;
using System.Linq;
public class Solution {
    
    Dictionary<string,List<string>> dic = new Dictionary<string,List<string>>();
    
    public int solution(string[,] clothes) {
        InsertToDictionary(clothes);
        return MakeAnswer();
    }
    
    private void InsertToDictionary(string[,] clothes){
        int rowNum = clothes.GetLength(0);
        for (int i=0; i<rowNum ;i++){
            string item = clothes[i,0];
            string type = clothes[i,1];
            if (!dic.ContainsKey(type)){
                dic[type] = new List<string>();
            }
            var list = dic[type];
            list.Add(item);
        }
        return;
    }
    
    private int MakeAnswer(){
        int answer=1;
        
        foreach (var pair in dic){
            var list = pair.Value;
            answer *= list.Count + 1;
        }
        
        return answer - 1;
    }
}
```
접근 방식도 좋았고, 과정도 좋았는데 아직 c#이 익숙치 않아서 콜렉션 활용이 좀 힘들었다. 

입력이 **string[,]** 으로 들어오는데, 기본적으로 선형구조의 배열이라고 생각해야한다. c의 2차원 배열이라고 생각하면 되겠다.

처음 접근시 foreach로 string[,]을 string[]으로 나눠서 구현했는데, string[,]은 선형구조의 일반적인 string[]와 같은 타입이기때문에 나눴더니 char형이 나와서 좀 당황했다.

그리고 HashMap을 찾고 있었는데, 여기는 Dictionary로 통한다하여 또 당황했다. 파이썬이랑 거의 비슷한 느낌인거 같다 아직까진,,

### 아쉬운 부분 최적화한다면..
현재 코드에서는 dictionary 타입이 <string,List<string>>
으로 실제 데이터 item을 list에 관리하고 있다. 하지만, 사용처가 list.Count 용도 100%이기 때문에 최적화를 한다면 처음부터 Count용도의 int로 value를 구성하여 구현하면 되겠다.

### 필요한 데이터만 활용하자
```c#
Dictionary<string, int> dic = new Dictionary<string, int>();

private void InsertToDictionary(string[,] clothes){
    int rowNum = clothes.GetLength(0);
    for (int i = 0; i < rowNum; i++){
        string type = clothes[i, 1];
        
        if (!dic.ContainsKey(type))
            dic[type] = 0;
        
        dic[type]++;
    }
}
```
